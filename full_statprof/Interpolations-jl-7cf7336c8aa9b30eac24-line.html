190 """
191 wi = WeightedIndex(indexes, weights)
192
193 Construct a weighted index `wi`, which can be thought of as a generalization of an
194 ordinary array index to the context of interpolation.
195 For an ordinary vector `a`, `a[i]` extracts the element at index `i`.
196 When interpolating, one is typically interested in a range of indexes and the output is
197 some weighted combination of array values at these indexes.
198 For example, for linear interpolation at a location `x` between integers `i` and `i+1`, we have
199
200 ret = (1-f)*a[i] + f*a[i+1]
201
202 where `f = x-i` lies between 0 and 1. This can be represented as `a[wi]`, where
203
204 wi = WeightedIndex(i:i+1, (1-f, f))
205
206 i.e.,
207
208 ret = sum(a[indexes] .* weights)
209
210 Linear interpolation thus constructs weighted indices using a 2-tuple for `weights` and
211 a length-2 `indexes` range.
212 Higher-order interpolation would involve more positions and weights (e.g., 3-tuples for
213 quadratic interpolation, 4-tuples for cubic).
214
215 In multiple dimensions, separable interpolation schemes are implemented in terms
216 of multiple weighted indices, accessing `A[wi1, wi2, ...]` where each `wi` is the
217 `WeightedIndex` along the corresponding dimension.
218
219 For value interpolation, `weights` will typically sum to 1.
220 However, for gradient and Hessian computation this will not necessarily be true.
221 For example, the gradient of one-dimensional linear interpolation can be represented as
222
223 gwi = WeightedIndex(i:i+1, (-1, 1))
224 g1 = a[gwi]
225
226 For a three-dimensional array `A`, one might compute `∂A/∂x₂` (the second component
227 of the gradient) as `A[wi1, gwi2, wi3]`, where `wi1` and `wi3` are "value" weights
228 and `gwi2` "gradient" weights.
229
230 `indexes` may be supplied as a range or as a tuple of the same length as `weights`.
231 The latter is applicable, e.g., for periodic boundary conditions.
232 """
233 abstract type WeightedIndex{L,W} end
