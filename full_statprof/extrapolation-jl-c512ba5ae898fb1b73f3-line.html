25 """
26 `extrapolate(itp, scheme)` adds extrapolation behavior to an interpolation object, according to the provided scheme.
27
28 The scheme can take any of these values:
29
30 * `Throw` - throws a BoundsError for out-of-bounds indices
31 * `Flat` - for constant extrapolation, taking the closest in-bounds value
32 * `Line` - linear extrapolation (the wrapped interpolation object must support gradient)
33 * `Reflect` - reflecting extrapolation (indices must support `mod`)
34 * `Periodic` - periodic extrapolation (indices must support `mod`)
35
36 You can also combine schemes in tuples. For example, the scheme `(Line(), Flat())` will use linear extrapolation in the first dimension, and constant in the second.
37
38 Finally, you can specify different extrapolation behavior in different direction. `((Line(),Flat()), Flat())` will extrapolate linearly in the first dimension if the index is too small, but use constant etrapolation if it is too large, and always use constant extrapolation in the second dimension.
39 """
