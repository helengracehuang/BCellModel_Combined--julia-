312 """
313 zeromatrix(u::AbstractVector)
314
315 Creates the zero'd matrix version of `u`. Note that this is unique because
316 `similar(u,length(u),length(u))` returns a mutable type, so it is not type-matching,
317 while `fill(zero(eltype(u)),length(u),length(u))` doesn't match the array type,
318 i.e., you'll get a CPU array from a GPU array. The generic fallback is
319 `u .* u' .* false`, which works on a surprising number of types, but can be broken
320 with weird (recursive) broadcast overloads. For higher-order tensors, this
321 returns the matrix linear operator type which acts on the `vec` of the array.
322 """
323 function zeromatrix(u)
324 x = safevec(u)
325 x .* x' .* false
326 end
