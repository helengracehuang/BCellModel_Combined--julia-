3 """
4 NamedTuple
5
6 `NamedTuple`s are, as their name suggests, named [`Tuple`](@ref)s. That is, they're a
7 tuple-like collection of values, where each entry has a unique name, represented as a
8 [`Symbol`](@ref). Like `Tuple`s, `NamedTuple`s are immutable; neither the names nor the values
9 can be modified in place after construction.
10
11 Accessing the value associated with a name in a named tuple can be done using field
12 access syntax, e.g. `x.a`, or using [`getindex`](@ref), e.g. `x[:a]` or `x[(:a, :b)]`.
13 A tuple of the names can be obtained using [`keys`](@ref), and a tuple of the values
14 can be obtained using [`values`](@ref).
15
16 !!! note
17 Iteration over `NamedTuple`s produces the *values* without the names. (See example
18 below.) To iterate over the name-value pairs, use the [`pairs`](@ref) function.
19
20 The [`@NamedTuple`](@ref) macro can be used for conveniently declaring `NamedTuple` types.
21
22 # Examples
23 ```jldoctest
24 julia> x = (a=1, b=2)
25 (a = 1, b = 2)
26
27 julia> x.a
28 1
29
30 julia> x[:a]
31 1
32
33 julia> x[(:a,)]
34 (a = 1,)
35
36 julia> keys(x)
37 (:a, :b)
38
39 julia> values(x)
40 (1, 2)
41
42 julia> collect(x)
43 2-element Vector{Int64}:
44 1
45 2
46
47 julia> collect(pairs(x))
48 2-element Vector{Pair{Symbol, Int64}}:
49 :a => 1
50 :b => 2
51 ```
52
53 In a similar fashion as to how one can define keyword arguments programmatically,
54 a named tuple can be created by giving a pair `name::Symbol => value` or splatting
55 an iterator yielding such pairs after a semicolon inside a tuple literal:
56
57 ```jldoctest
58 julia> (; :a => 1)
59 (a = 1,)
60
61 julia> keys = (:a, :b, :c); values = (1, 2, 3);
62
63 julia> (; zip(keys, values)...)
64 (a = 1, b = 2, c = 3)
65 ```
66
67 As in keyword arguments, identifiers and dot expressions imply names:
68
69 ```jldoctest
70 julia> x = 0
71 0
72
73 julia> t = (; x)
74 (x = 0,)
75
76 julia> (; t.x)
77 (x = 0,)
78 ```
79
80 !!! compat "Julia 1.5"
81 Implicit names from identifiers and dot expressions are available as of Julia 1.5.
82
83 !!! compat "Julia 1.7"
84 Use of `getindex` methods with multiple `Symbol`s is available as of Julia 1.7.

228 """
229 merge(a::NamedTuple, bs::NamedTuple...)
230
231 Construct a new named tuple by merging two or more existing ones, in a left-associative
232 manner. Merging proceeds left-to-right, between pairs of named tuples, and so the order of fields
233 present in both the leftmost and rightmost named tuples take the same position as they are found in the
234 leftmost named tuple. However, values are taken from matching fields in the rightmost named tuple that
235 contains that field. Fields present in only the rightmost named tuple of a pair are appended at the end.
236 A fallback is implemented for when only a single named tuple is supplied,
237 with signature `merge(a::NamedTuple)`.
238
239 !!! compat "Julia 1.1"
240 Merging 3 or more `NamedTuple` requires at least Julia 1.1.
241
242 # Examples
243 ```jldoctest
244 julia> merge((a=1, b=2, c=3), (b=4, d=5))
245 (a = 1, b = 4, c = 3, d = 5)
246 ```
247
248 ```jldoctest
249 julia> merge((a=1, b=2), (b=3, c=(d=1,)), (c=(d=2,),))
250 (a = 1, b = 3, c = (d = 2,))
251 ```

281 Interpret an iterable of key-value pairs as a named tuple, and perform a merge.
282
283 ```jldoctest
284 julia> merge((a=1, b=2, c=3), [:b=>4, :d=>5])
285 (a = 1, b = 4, c = 3, d = 5)
286 ```

288 function merge(a::NamedTuple, itr)
289 names = Symbol[]
290 vals = Any[]
291 inds = IdDict{Symbol,Int}()
292 for (k, v) in itr
293 k = k::Symbol
294 oldind = get(inds, k, 0)
295 if oldind > 0
296 vals[oldind] = v
297 else
298 push!(names, k)
299 push!(vals, v)
300 inds[k] = length(names)
301 end
302 end

371 """
372 @NamedTuple{key1::Type1, key2::Type2, ...}
373 @NamedTuple begin key1::Type1; key2::Type2; ...; end
374
375 This macro gives a more convenient syntax for declaring `NamedTuple` types. It returns a `NamedTuple`
376 type with the given keys and types, equivalent to `NamedTuple{(:key1, :key2, ...), Tuple{Type1,Type2,...}}`.
377 If the `::Type` declaration is omitted, it is taken to be `Any`. The `begin ... end` form allows the
378 declarations to be split across multiple lines (similar to a `struct` declaration), but is otherwise
379 equivalent.
380
381 For example, the tuple `(a=3.1, b="hello")` has a type `NamedTuple{(:a, :b),Tuple{Float64,String}}`, which
382 can also be declared via `@NamedTuple` as:
383
384 ```jldoctest
385 julia> @NamedTuple{a::Float64, b::String}
386 NamedTuple{(:a, :b), Tuple{Float64, String}}
387
388 julia> @NamedTuple begin
389 a::Float64
390 b::String
391 end
392 NamedTuple{(:a, :b), Tuple{Float64, String}}
393 ```
394
395 !!! compat "Julia 1.5"
396 This macro is available as of Julia 1.5.
397 """
398 macro NamedTuple(ex)
399 Meta.isexpr(ex, :braces) || Meta.isexpr(ex, :block) ||
400 throw(ArgumentError("@NamedTuple expects {...} or begin...end"))
401 decls = filter(e -> !(e isa LineNumberNode), ex.args)
402 all(e -> e isa Symbol || Meta.isexpr(e, :(::)), decls) ||
403 throw(ArgumentError("@NamedTuple must contain a sequence of name or name::type expressions"))
404 vars = [QuoteNode(e isa Symbol ? e : e.args[1]) for e in decls]
405 types = [esc(e isa Symbol ? :Any : e.args[2]) for e in decls]
406 return :(NamedTuple{($(vars...),), Tuple{$(types...)}})
